# Feature: 5.3.1 RDF Infrastructure Setup

## Problem Statement

The Knowledge Graph Foundation needs RDF-based infrastructure for future context management. This task creates placeholder modules and basic data structures using RDF.ex without implementing full functionality. The goal is to establish the namespace, verify RDF.ex works correctly, and create stub functions for future implementation.

## Solution Overview

Create the following module structure:
1. `JidoCode.KnowledgeGraph` - Namespace module with documentation
2. `JidoCode.KnowledgeGraph.Store` - RDF.Graph wrapper with stub functions
3. `JidoCode.KnowledgeGraph.Vocab.Code` - Custom vocabulary for code entities
4. `JidoCode.KnowledgeGraph.Entity` - Struct for code entities

## Technical Details

### Dependencies
- `rdf` ~> 2.0 (already in mix.exs, version 2.1.0 locked)
- `libgraph` ~> 0.16 (already in mix.exs, for task 5.3.2)

### Files to Create
- `lib/jido_code/knowledge_graph.ex` - Namespace module
- `lib/jido_code/knowledge_graph/store.ex` - RDF.Graph wrapper
- `lib/jido_code/knowledge_graph/vocab/code.ex` - Code vocabulary
- `lib/jido_code/knowledge_graph/entity.ex` - Entity struct
- `test/jido_code/knowledge_graph/store_test.exs` - Store tests
- `test/jido_code/knowledge_graph/entity_test.exs` - Entity tests

### Vocabulary Design

The Code vocabulary will define terms for Elixir code entities:
- Entity types: `Module`, `Function`, `Type`, `Protocol`, `Behaviour`
- Relationships: `defines`, `calls`, `imports`, `uses`, `implements`
- Properties: `name`, `arity`, `visibility`, `doc`

Base IRI: `https://jidocode.dev/vocab/code/`
