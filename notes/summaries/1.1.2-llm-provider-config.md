### JidoCode.Config Module

**Public API:**
- `get_llm_config/0` - Returns `{:ok, config}` or `{:error, reason}`
- `get_llm_config!/0` - Returns config or raises RuntimeError
- `configured?/0` - Returns boolean

**Configuration Schema:**
```elixir
%{
  provider: :anthropic,           # Required atom
  model: "claude-3-5-sonnet",     # Required string
  temperature: 0.7,               # Optional float (default: 0.7)
  max_tokens: 4096                # Optional integer (default: 4096)
}
```

**Features:**
- Validates provider exists via `Jido.AI.Provider.providers/0` (50+ providers)
- Validates API key via environment variable or JidoAI Keyring fallback
- Supports environment variable overrides:
  - `JIDO_CODE_PROVIDER` overrides `:provider`
  - `JIDO_CODE_MODEL` overrides `:model`
- Provider-specific API keys: `ANTHROPIC_API_KEY`, `OPENAI_API_KEY`, etc.
- Handles Keyring GenServer not running (catches :exit)

### Error Messages

Clear, actionable error messages:
- "No LLM provider configured. Set JIDO_CODE_PROVIDER or configure :jido_code, :llm, :provider"
- "Invalid provider 'X'. Available providers include: anthropic, openai... (50+ total)"
- "No API key found for provider 'X'. Set X_API_KEY environment variable."
