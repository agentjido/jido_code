# Summary: Task 1.3.4 - Provider and Model Lists

## Overview

Implemented `get_providers/0` and `get_models/1` functions that return user-configured lists from settings, falling back to JidoAI dynamic discovery when not configured.

## Changes Made

### Modified Files
- `lib/jido_code/settings.ex` - Added get_providers/0, get_models/1 functions (~60 lines)
- `test/jido_code/settings_test.exs` - Added 9 new tests

### JidoCode.Settings Module Updates

**New Public API:**
- `get_providers/0` - Returns providers list from settings or JidoAI fallback
- `get_models/1` - Returns models list for a provider from settings or JidoAI fallback

**Fallback Logic:**
```elixir
# get_providers/0 priority:
# 1. Settings "providers" list (if non-empty)
# 2. Jido.AI.Provider.providers/0 (dynamic discovery)

# get_models/1 priority:
# 1. Settings "models[provider]" list (if non-empty)
# 2. Jido.AI.Provider.list_all_models_enhanced/1 (dynamic discovery)
```

**JidoAI Integration:**
- Converts provider atoms to strings for consistent return type
- Uses `String.to_existing_atom/1` for safe atom creation
- Extracts model IDs from enhanced model structs
- Gracefully handles errors with try/rescue/catch (returns empty list)

### Tests Added (9 tests)

**get_providers/0 tests:**
- Returns list of strings
- Falls back to JidoAI when settings empty
- Returns settings list when configured
- Handles empty settings providers list

**get_models/1 tests:**
- Returns list of strings for valid provider
- Returns empty list for unknown provider
- Returns settings list when configured
- Falls back to JidoAI when settings empty
- Handles empty settings models list

## Verification

- `mix test` - 105 tests, 0 failures (9 new)

## Usage Example

```elixir
# Get providers (settings or JidoAI fallback)
JidoCode.Settings.get_providers()
#=> ["anthropic", "openai", "google", ...]

# Get models for provider (settings or JidoAI fallback)
JidoCode.Settings.get_models("anthropic")
#=> ["claude-3-5-sonnet-20241022", "claude-3-opus-20240229", ...]

# Unknown provider returns empty list
JidoCode.Settings.get_models("unknown_provider")
#=> []
```

## JidoAI API Notes

During implementation, discovered:
- `Jido.AI.Provider.providers/0` returns keyword list with atoms as keys
- `Jido.AI.Provider.list_all_models_enhanced/1` returns `{:ok, models}` tuple
- Model structs have `:id` and `:name` fields for extracting model identifiers
