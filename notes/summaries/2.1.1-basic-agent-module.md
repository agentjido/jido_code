# Summary: Task 2.1.1 Basic Agent Module

## Overview

Implemented `JidoCode.Agents.LLMAgent`, a GenServer wrapper around JidoAI's Agent system that provides chat functionality with configurable LLM providers.

## Changes Made

### New Files

1. **`lib/jido_code/agents/llm_agent.ex`**
   - GenServer wrapping `Jido.AI.Agent`
   - `start_link/1` - Starts agent with config from JidoCode.Config or explicit options
   - `chat/2` - Sends message to LLM and broadcasts response to PubSub
   - `get_config/1` - Returns current model configuration
   - System prompt defining coding assistant personality
   - Exit trapping for graceful AI agent crash handling

2. **`test/jido_code/agents/llm_agent_test.exs`**
   - 7 tests (6 pass, 1 integration test skipped)
   - Tests for start_link with various configs
   - Tests for configuration override
   - Tests for get_config

### Key Implementation Details

- Uses `Jido.AI.Model.from/1` to build model from provider tuple
- System prompt uses EEx template with `@message` variable
- Broadcasts responses to `"tui.events"` PubSub topic
- Handles AI agent exits with `Process.flag(:trap_exit, true)`
- Falls back to explicit opts if JidoCode.Config is not configured

## Test Results

```
129 tests, 0 failures, 1 skipped
```

The skipped test (`chat/2 sends message and receives response`) is an integration test that requires a real API key. Run with `mix test --include integration` when API key is available.

## API Usage

```elixir
# Start with default config
{:ok, pid} = JidoCode.Agents.LLMAgent.start_link()

# Start with custom options
{:ok, pid} = JidoCode.Agents.LLMAgent.start_link(
  provider: :anthropic,
  model: "claude-3-5-sonnet-20241022",
  temperature: 0.5
)

# Send chat message
{:ok, response} = JidoCode.Agents.LLMAgent.chat(pid, "How do I reverse a list in Elixir?")

# Get current config
config = JidoCode.Agents.LLMAgent.get_config(pid)
```
