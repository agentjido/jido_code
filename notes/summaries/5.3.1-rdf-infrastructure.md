# Summary: 5.3.1 RDF Infrastructure Setup

## Overview

Created the foundation modules for RDF-based knowledge representation using RDF.ex. This establishes placeholder infrastructure for future context management in the knowledge graph.

## Files Created

### lib/jido_code/knowledge_graph.ex
- Namespace module with documentation
- `base_iri/0` returns "https://jidocode.dev/vocab/code/"
- `version/0` returns "0.1.0"

### lib/jido_code/knowledge_graph/vocab/code.ex
- `JidoCode.KnowledgeGraph.Vocab` module using RDF.Vocabulary.Namespace
- Defines `Code` vocabulary with defvocab macro
- Entity types: Module, Function, Type, Protocol, Behaviour, Macro, Struct, Exception
- Relationships: defines, calls, imports, uses, implements, depends_on, supervises, aliases
- Properties: name, arity, visibility, doc, file_path, line_number, module_name, spec
- Base IRI: https://jidocode.dev/vocab/code/

### lib/jido_code/knowledge_graph/entity.ex
- `JidoCode.KnowledgeGraph.Entity` struct
- Fields: type, name, module, arity, visibility, file_path, line_number, doc, metadata
- `new/3` - Creates entities with options
- `qualified_name/1` - Returns fully qualified name (e.g., "MyApp.User.create/1")
- `to_iri/1` - Generates RDF IRI for the entity

### lib/jido_code/knowledge_graph/store.ex
- `JidoCode.KnowledgeGraph.Store` struct wrapping RDF.Graph
- Working functions: `new/1`, `empty?/1`, `count/1`, `to_graph/1`, `add_triple/2`
- Stub functions (return `{:error, :not_implemented}`): `add_entity/2`, `query/2`, `clear/1`

## Technical Notes

### RDF.ex Vocabulary Access
- Lowercase terms accessed directly: `Vocab.Code.defines()`
- Capitalized terms accessed via apply: `apply(Vocab.Code, :Module, [])`
- This is due to Elixir treating capitalized names as module references
